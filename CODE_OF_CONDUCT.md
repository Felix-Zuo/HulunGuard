# Code of Conduct

HulunGuard is an engineering project for reliability, evidence, and safer agent workflows. Project spaces should stay focused, respectful, and useful.

## Expected Behavior

- Be direct without being hostile.
- Critique code, claims, tests, and product tradeoffs rather than people.
- Provide reproducible details for bugs, security issues, and adapter behavior.
- Respect privacy boundaries. Do not post private prompts, credentials, customer data, production logs, or sensitive traces.
- Assume maintainers will prioritize safety, correctness, and release quality over speed.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private data, credentials, secrets, or non-consensual logs.
- Repeated off-topic comments that block maintainers or contributors from doing project work.
- Deliberately misleading reports, fabricated evidence, or attempts to bypass security and release gates.

## Enforcement

Maintainers may edit or remove comments, close issues, restrict participation, or report abuse when behavior harms the project or its users.

For security-sensitive reports, use the process in `SECURITY.md`. For normal project issues, use the GitHub issue templates and include reproducible evidence.
